## Vendor Partnership Model

Every hardware recommendation in the platform is a revenue opportunity. The model:

### Affiliate / Commission Structure

| Vendor Type | Examples | Commission Model |
|-------------|----------|-----------------|
| Pixel + strip suppliers | HolidayCoro, Ray Wu | 5-10% affiliate commission |
| Controller manufacturers | Falcon, HinksPix, Kulp | 5-8% or store credit referral |
| Prop makers (coro) | Boscoyo, HolidayCoro | 5-10% affiliate |
| HD prop makers | Gilbert Engineering | Partnership / co-marketing |
| General hardware | Amazon | Amazon Associates (3-4%) |
| Power supplies | Amazon, specialty | Amazon Associates |

### Value Proposition to Vendors

"Lights of Elm Ridge sends you qualified, ready-to-buy customers with a specific shopping list. They know exactly what they need, in the right quantities, from your store. No tire-kicking."

What you offer vendors:
- **Pre-qualified traffic**: Users arrive at vendor sites with a specific shopping list, not browsing. Conversion rates will be high.
- **Reduced support burden**: Users who go through the program won't email vendors asking "what do I need?" â€” they already know.
- **Co-branded content**: "Recommended by Lights of Elm Ridge" badge on vendor product pages.
- **Data**: "We sent X users to your store this month, they purchased $Y on average."

### Revenue Tiers

| Tier | Model | Revenue |
|------|-------|---------|
| Free tier | Orientation video, basic shop wizard | Free (top of funnel) |
| Sequence purchases | Individual sequences + catalog | $15-30 per sequence |
| xWire | Wiring diagram generation | $10-20 per layout or bundled with first sequence purchase |
| Mod:IQ | Included free with every sequence | Drives sequence sales volume |
| Vendor commissions | Affiliate links on all hardware | 3-10% of referred sales |
| Premium support | "Review my wiring plan" / custom help | $50-100 per session |

### Revenue Math (Conservative)

If 1,000 new hobbyists/year go through the program:
- Average hardware spend: $2,000 Ã— 7% blended commission = **$140/user** = **$140K/year**
- Average first-year sequence spend: $150 Ã— 1,000 = **$150K/year**
- xWire: $15 Ã— 1,000 = **$15K/year**
- Year 2+ retention (60%): 600 users Ã— $100 sequences = **$60K/year** (compounds)

**Year 1: ~$305K** | **Year 2: ~$365K** | **Year 3: ~$400K+**

And this is conservative â€” it assumes 1,000 users/year through the full program. The onboarding platform itself could grow the hobby's conversion rate, meaning more than the current ~3,000 new hobbyists/year actually make it through.

## Layout File Generation: Shop Wizard â†’ xLights

The killer handoff: when a user finishes the Shop Wizard, they download an `.rgbeffects` file that opens directly in xLights with their entire display pre-built. No manual model creation. No guessing at pixel counts or channel assignments. They open xLights and their display is there.

### What the .rgbeffects File Contains

The .rgbeffects file is XML. Each model definition includes:

```xml
<model name="Arch 1" DisplayAs="Arches" StringType="RGB Nodes"
       parm1="25" parm2="1" parm3="1"
       StartSide="B" Dir="L" Antialias="1"
       PixelCount="25" PixelType="Square"
       StartChannel=">!Controller:1:1"
       LayoutGroup="Default"
       WorldPosX="2.5" WorldPosY="1.0" WorldPosZ="0.0"
       ScaleX="1.0" ScaleY="0.5" />
```

Key attributes per model:
- **DisplayAs**: Model type (Arches, Tree, Candy Canes, Matrix, Custom, Spinner, etc.)
- **parm1/2/3**: Type-specific parameters (pixel count, string count, strands, etc.)
- **StartChannel**: Controller port assignment using xLights reference format
- **WorldPos/Scale**: 2D layout positioning
- **StringType**: Always "RGB Nodes" for pixel props

The second file to generate. This defines the controller hardware so the user doesn't have to touch the Controllers tab at all.

**Schema:**
```xml

         AutoUpdateFromBase="0" BaseShowDir="">

             NetworkType="E131" MaxChannels="510"/>
    <network ComPort="192.168.1.100" BaudRate="2"

  </Controller>

```

**Key fields per controller:**
- `Vendor` + `Model`: Must match xLights' known controller list (HinksPix PRO V3, Falcon F48, AlphaPix 16, etc.)
- `IP`: Default to `192.168.1.100` â€” user changes this to match their actual controller IP. Call out in video.
- `Protocol`: E131 for most setups (DDP is an option but E131 is more beginner-friendly)
- `AutoSize="1"` + `AutoLayout="1"`: Let xLights auto-manage channel allocation â€” critical for beginners
- `FullxLightsControl="TRUE"`: xLights manages the controller config directly
